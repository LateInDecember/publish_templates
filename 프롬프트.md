## 에이전트에게 보내는 지시

당신은 **Quarto + Obsidian + Zotero + R**(또는 다른 분석 도구) 기반의 학술 논문 원고 작성·렌더링 환경을 처음부터 구축합니다. 아무 사전 지식이 없는 사람의 새 컴퓨터에서도 이 프롬프트만으로 재현되어야 합니다. **Phase 0 → 7**을 순서대로 수행하고, 각 Phase 끝에 한 줄로 보고합니다. **임의로 폴더를 만들거나 순서를 건너뛰지 마세요.**

### Phase 0 — 사용자에게 질문 (먼저, 한 번에)

`setup_questions.md`의 5가지를 묻고 답을 받습니다: ① 작업 폴더 위치+이름, ② 분석 데이터/결과 위치(없으면 "아직 없음"), ③ 문헌·참고문헌 위치(Zotero 컬렉션/PDF 폴더/references.bib), ④ 원고 양식(없으면 기본 양식 사용), ⑤ 분석 도구(기본 R). 답 전에는 파일을 만들지 않습니다.

### Phase 1 — 작업 폴더 + 폴더 구조 생성

`folder_structure.md`대로 `01_manuscript/`(4구역 + `_scripts`/`_logs`/`_archive`)와 `02_anal/`을 생성. 번호 폴더=콘텐츠, 언더스코어=부산물. csv·로그는 항상 `_logs/`. 빈/타임스탬프 폴더 남발 금지.

> **자동**: 프로젝트 경로를 받은 뒤 `bash install/bootstrap.sh "<프로젝트경로>" [--install]` 한 번으로 Phase 1(폴더) + Phase 4(양식·스크립트·.obsidian·AGENTS) + 검증을 처리할 수 있다. 이후 GUI 연동(Zotero/Obsidian)·프로젝트별 값만 마무리.

### Phase 2 — 최상위 AGENTS.md 생성

`AGENTS_TEMPLATE.md`의 `<...>`를 실제 값으로 치환해 프로젝트 루트에 `AGENTS.md`로 저장. 이후 작업 규칙 정본.

### Phase 3 — Quarto + Zotero + Obsidian (+R) 설치

`install_quarto_zotero_obsidian.md`를 따름:
- **Quarto** 설치 → `quarto --version`.
- **Zotero + Better BibTeX** → 컬렉션을 `01_source/references.bib`로 자동 export(Keep updated).
- **Obsidian** → `01_manuscript/`를 vault로 열기. 원고는 `manuscript.md`를 편집(=Quarto 소스, 별도 `.qmd` 없음). Obsidian 안 렌더는 Shell commands 플러그인에 `Rscript _scripts/render_with_insertions.R` 바인딩 → `obsidian_workflow.md` 참조.
- **R 필수** — `install.sh`/`install.ps1`(또는 bootstrap 기본 실행)이 설치. 원고 렌더 `render_with_insertions.R`가 R 기반 + 패키지 `officer,png,stringr,xml2,zip` 필요. 분석 자체는 Python/MATLAB도 가능하고 `02_anal`은 도구 중립 구조(`01_R` 강제 없음)지만, **렌더용 R은 반드시 설치**한다.

> **설치는 직접 실행**한다. "설치하세요"라고 말만 하고 넘어가면 파일만 clone되고 도구는 안 깔린다. 로컬 셸이 있으면 `bash install/install.sh`(또는 bootstrap 기본)를 **에이전트가 실제로 실행**한다. 셸이 격리/원격이라 사용자 컴퓨터에 설치 불가하면, 건너뛰지 말고 사용자에게 한 줄 명령을 주어 실행하게 하고 완료를 확인한다.

### Phase 4 — 원고 양식 세팅

`manuscript_format/`로 골격 구성: `_quarto.yml`(html `embed-resources: true` 유지) → `01_manuscript/`, `styles/` → `01_source/styles/`, 양식 노트 → `01_source/notes/`, `manuscript_skeleton.md` → `manuscript.md`, `scripts/*` → `_scripts/`(+ `render.command` → `01_source/`, `chmod +x`). **표·그림 마커 매핑을 새 프로젝트 실제 표/그림에 맞게 수정**해야 함을 명시. 사용자가 자기 양식을 주면 그것으로 대체(마커·폴더 규칙·embed-resources 유지).

### Phase 5 — 데이터·문헌 연결

- 분석 결과: `sync_reporting_assets.R`의 `reporting_root`를 사용자 실제 분석 폴더로. 없으면 폴더만 두고 안내.
- 문헌: Zotero→`references.bib` 자동 export 확인 → `sync_zotero_literature.R`로 `02_literature/`에 미러. 로컬 PDF 폴더는 `sync_key_articles_to_literature.R`의 `source_root`로.
- **Zotero API 키**: `_secrets/`에 보관. `bash _secrets/set_zotero_key.sh`로 복붙 저장(gitignore·권한600). 에이전트는 `_secrets/zotero.env`를 **읽기 전용**으로만 쓰고 키를 출력·커밋하지 않는다.
- **문헌 노트**: `02_literature/pdfs/`의 논문들을 `write-literature-note` 스킬(또는 `literature_note_template.md` 양식)로 **17섹션 paper-review 노트**로 작성. 프론트매터 `status`로 core/candidate/dropped 분류.

### Phase 6 — 렌더 검증

분석 실행 → `sync_reporting_assets.R --dry-run` → `render.command`(또는 `Rscript _scripts/render_with_insertions.R`) → `05_output/manuscript.docx`, `.html` 확인. `05_output`에 그림/스타일 복사 폴더가 생기면 `_quarto.yml` html의 `embed-resources: true` 확인. 렌더 전 Word에서 docx 닫기.

### Phase 7 — 마무리 보고

구조·도구 버전·연결 경로·렌더 성공 여부 요약 + 미완 체크리스트(마커 매핑 수정, 분석 미실행 등).
